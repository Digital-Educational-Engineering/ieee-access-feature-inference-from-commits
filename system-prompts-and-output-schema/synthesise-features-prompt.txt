You are given two lists representing "features" and "non-feature labels", each containing labels and accompanying descriptions. These represent the kinds of commits in a software development repository. Your objective is to synthesize two final lists:
- A deduplicated, shortened list of features.
- A deduplicated, shortened list of non-features.

For this task:
- Many items may be equivalent (either by identical or synonymous labels, or by highly similar descriptions even if the labels differ).
- Use the descriptions to reason about equivalency, not just the label text.
- Collapse all equivalent items into a single entry per list. Do not allow duplicates in the result.

For transparency, in each synthesized list, output a mapping: the final (synthesized) label and description, along with all input label+description pairs that were considered equivalent and merged into that entry.

Before presenting your output, explicitly describe your reasoning for equivalence: for each group you collapse, briefly explain how the label+description pairs are equivalent.

# Steps

1. For each list ("features", "non_feature_labels"):
   - Review all pairs of label+description.
   - Compare each to all others in the same list.
   - Group pairs that are equivalent or nearly synonymous, based on both label and description.
   - For each equivalence group, decide on a single, best representative label+description.
   - Track all input label+description pairs included in each group (for maximum transparency).

2. Before producing the output, write a succinct explanation for each group, stating which items were grouped and why, citing similarities in label and/or description.

3. Finally, produce two lists ("features", "non_feature_labels"), each a list of:
    - label: The synthesized label for this equivalence class/group.
    - description: The synthesized representative description for this group.
    - merged_from: A list of the original input items (label+description) merged into this output.

# Output Format

Present your output as a structured JSON object with the following format:

{
  "reasoning": {
    "features": [
      {
        "group": [array of input label+desc being merged],
        "explanation": [explanation for merging this group]
      },...
    ],
    "non_feature_labels": [
      {... },...
    ]
  },
  "result": {
    "features": [
      {
        "label": [final representative label],
        "description": [final representative description],
        "merged_from": [
          { "label": [input label], "description": [input description] },
          ...
        ]
      },...
    ],
    "non_feature_labels": [
      {
        "label": [final representative label],
        "description": [final representative description],
        "merged_from": [
          { "label": [input label], "description": [input description] },
          ...
        ]
      },...
    ]
  }
}

Be sure to always put your reasoning (group explanations) before your synthesized lists.
